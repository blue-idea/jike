## 📖 项目简介

**集刻**是一款以中国文化旅游为核心的精美移动应用。依托**全国 A 级景点名录**、**全国博物馆名录**、**全国重点文物保护单位名录**三大权威数据源，结合**高德地图**的空间定位能力与**AI 大模型**的文化解析能力，让每一次旅行都成为一段沉浸式的文化之旅。

> 不只是旅游，更是一种文化生活方式的养成。

---

## ✨ 核心功能

### 🗺️ 首页 · 发现 (Discovery)

- 大型柔焦历史古迹 Hero Banner，极简文字标题
- 个性化文化推荐轮播：根据用户偏好推送精选合集（如"宋代隐藏古刹"、"江南明清园林"）
- 全国文化地图概览，快速定位感兴趣的文化区域
- 编辑精选专题：节气文化、朝代建筑、石窟艺术等主题合集

### 🔍 搜索与 POI 详情 (Search & Heritage Detail)

- 全文搜索景区、博物馆、国保单位
- 多维文化标签筛选：朝代（唐 / 宋 / 明 / 清）× 类型（古建筑 / 石窟 / 陵墓 / 战役遗址）
- 详情页强调故事叙述：历史照片、优雅排版、用户游记
- 展示文物时代背景、文化价值、开放信息、周边交通

### 🗾 交互地图 (Interactive Map)

- 基于**高德地图**的自定义文旅皮肤：弱化现代道路，突出古建筑与文博点位
- 全国 A 级景点、博物馆、国保单位精美发光点位标记
- **附近文化地标**：自动定位，过滤非文化类 POI，仅展示博物馆、国保单位及 4A-5A 景区
- 支持"附近 500 米内的文化遗址"精准搜索
- 热力图：展示热门景点人流趋势

### 📍 路线规划 (Route Planning)

- 拖拽式路线安排，集成地图规划界面
- 支持步行 / 驾车 / 公交多种出行方式
- 多点路径智能优化（旅游路线串联）
- 一键生成区域文化散步路线（如"西安城墙内明清建筑路线"）
- 路径规划 API 自动串联周边国保单位

### 🤖 AI 增强 (AI-Powered Features)

#### 🎙️ AI 导游（核心卖点）
- AI 生成场馆、景点深度介绍：
  - 📜 历史背景与文化解读
  - 📖 相关诗词与人物故事
  - 🏛️ 朝代演变时间线
  - 👁️ 重要看点导览
- 支持语音播报，沉浸式讲解体验

#### 🧭 智能路线规划
- 自然语言输入：*"我喜欢历史 + 不走太累 + 2 天"*
- 输出定制路线 + 时间安排 + 文化主题标注

#### 📸 拍照识别
- 拍摄建筑 → 自动识别景点
- 即时输出文化信息与历史背景

#### 💬 文化知识问答
- 支持文化知识百科式问答
- 示例：*"中国四大石窟有哪些？"*、*"哪个朝代佛教最兴盛？"*

#### 📔 旅行日志自动生成
- 自动记录游览轨迹
- AI 辅助生成图文游记

### 🛂 数字文旅护照 (Digital Passport)

- **虚拟印章系统**：模拟线下盖章，设计精美数字艺术印章，到访即可解锁
- **地理围栏打卡**：基于高德 Geofencing，进入国保单位或博物馆 500 米范围内自动点亮图标
- **足迹热力图**：可视化展示用户在全国范围内点亮的文旅版图
- **文化养成进度条**：记录探索深度与广度
- **称号成就系统**：
  - 🏯 *宋代建筑狂热者*
  - 🪨 *石刻重度爱好者*
  - 🏛️ *博物馆资深玩家*
  - 更多称号等待解锁...

### 👤 个人中心 (Profile)

- 文化足迹地图全览
- 徽章与印章收藏展示
- 游览历史与统计数据
- 个人旅行日志管理

## 📊 数据来源

| 数据集 | 主要字段 | 规模 |
|--------|---------|------|
| **全国 A 级旅游景点名录** | 景区名称、等级（1A-5A）、省级行政区、编码地址、WGS84 经纬度 | 全国覆盖 |
| **全国重点文物保护单位** | 名称、文物类型、时代、批次、省/市/区县、经纬度、文化维度标签 | 国家级文保 |
| **全国博物馆名录** | 省份、博物馆名称、质量等级、性质、是否免费开放 | 全国覆盖 |

> 文保单位采用**文化维度标签**而非简单分类，例如：
> - 朝代维度：唐 / 宋 / 元 / 明 / 清
> - 类型维度：古建筑 / 石窟寺 / 陵墓 / 战役遗址 / 近现代史迹

## 🚀 快速开始

### 环境要求

- Node.js >= 18
- pnpm >= 8（推荐）或 npm
- Expo CLI
- iOS Simulator / Android Emulator 或 Expo Go App

### 安装依赖

```sh
jike/package.json#L1-1
pnpm install
```

### 启动开发服务器

```sh
jike/package.json#L1-1
pnpm dev
```

扫描终端中显示的二维码，在 **Expo Go** 中即可预览应用。

### 构建 Web 版本

```sh
jike/package.json#L1-1
pnpm build:web
```

### 类型检查

```sh
jike/package.json#L1-1
pnpm typecheck
```

### Git 提交前检查（Husky + lint-staged）

在仓库根目录执行 `npm install` 时，`prepare` 脚本会安装 [Husky](https://typicode.github.io/husky/) Git 钩子。`git commit` 触发 `pre-commit`：对暂存区匹配的 `*.ts`、`*.tsx`、`*.js`、`*.jsx`、`*.mjs`、`*.cjs` 运行 ESLint（`eslint --fix`）；若暂存包含 `.ts` / `.tsx`，再执行一次全项目 `npm run typecheck`（`tsc --noEmit`）。任一检查失败则阻断提交。

新克隆仓库后按上文安装依赖即可复现钩子；若未生效，请确认已执行 `npm install` 且存在 `.husky/pre-commit`。

仅在排障或紧急场景需要跳过钩子：`git commit --no-verify`（或 `git commit -n`）。

## ⚙️ 环境配置

在项目根目录创建 `.env.local` 文件，配置以下环境变量：

```/dev/null/.env.example#L1-8
# Supabase
EXPO_PUBLIC_SUPABASE_URL=your_supabase_url
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# 高德地图
EXPO_PUBLIC_AMAP_KEY=your_amap_api_key

# AI 服务
EXPO_PUBLIC_AI_API_KEY=your_ai_api_key
```
